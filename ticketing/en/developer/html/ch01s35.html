1.35. Data usage analytics

The usage analytics API lets you access information about how people are using DHIS 2based on data analysis. When users access favorites, an event is recorded. The event consists of the user name, the UID of the favorite, when the event took place, and the type of event. The different types of events are listed in the Table 1.67, “Supported event types” table.

This resource lets you retrieve aggregated snapshots of usage analytics based on time intervals. The resource captures user views (for example the number of times a chart or pivot table has been viewed by a user) and saved analysis favorites (for example favorite charts and pivot tables). When enabled, DHIS 2 will capture nightly snapshots which are then aggregated at request.

/api/dataStatistics

1.35.1. Request query parameter

The usage analytics (data statistics) API supports two operations:

  • POST: creates a view event

  • GET: retrieves aggregated statistics

1.35.2. Create view events (POST)

The usage analytics API lets you create event views. The dataStatisticsEventType parameter describes what type of item was viewed. The favorite parameter indicates the identifier of the relevant favorite.

Example 1.1. URL that creates a new event view of charts

POST /api/dataStatistics?eventType=CHART_VIEW&favorite=LW0O27b7TdD


A successful save operation returns HTTP status code 201. The table below shows the supported types of events.

Table 1.67. Supported event types

KeyDescription
REPORT_TABLE_VIEWReport table (pivot table) view
CHART_VIEWChart view
MAP_VIEWMap view (GIS)
EVENT_REPORT_VIEWEvent report view
EVENT_CHART_VIEWEvent chart view
DASHBOARD_VIEWDashboard view

1.35.3. Retrieve aggregated usage analytics report (GET)

The usage analytics (data statistics) API lets you specify certain query parameters when asking for an aggregated report.

Table 1.68. Query parameters for aggregated usage analytics (data statistics)

Query parameterRequiredDescriptionOptions
startDateYesStart date for periodDate in yyyy-MM-dd format
endDateYesEnd date for periodDate in yyyy-MM-dd format
intervalYesType of interval to be aggregatedDAY, WEEK, MONTH, YEAR


The startDate and endDate parameters specify the period for which snapshots should be used in the aggregation. You must format the dates as shown above. If no snapshots are saved in the specified period, an empty list is sent back. The interval parameter specifies what kind of aggregation will be done.

Example 1.2. URL that creates a query for a yearly aggregation

GET /api/dataStatistics?startDate=2014-01-02&endDate=2016-01-01&interval=MONTH


1.35.4. Response format

You can return the aggregated data in a usage analytics response in several representation formats. The default format is JSON. The available formats and content types are:

  • json (application/json)

  • xml (application/xml)

  • html (text/html)

Example 1.3. URL that requests an usage analytics response in XML format 

/api/dataStatistics.xml?startDate=2014-01-01&endDate=2016-01-01&interval=WEEK


You must retrieve the aggregated usage analytics response with the HTTP GET method. This allows you to link directly from Web pages and other HTTP-enabled clients to usage analytics responses . To do functional testing, use the URL library.

Example 1.4. Usage analytics response in JSON format

Execute this command against the demo database to get an usage analytics response in JSON format: 

curl "play.dhis2.org/demo/api/dataStatistics?startDate=2016-02-01&endDate=2016-02-14&

The response represents a list of aggregated data over two weekly intervals:

  • year: the year the aggregated data is from

  • week: the week number the aggregated data is from

  • averageViews: average number of views during the selected period

  • totalViews: total number of views during the selected period

  • activeUsers: maximum (peak) number of unique users that have made an event view during the selected period

  • users: total maximum (peak) number of users in the database during the selected period

  • The various views properties show the number of views of each event (for example map, chart) during the selected period.

  • The various saved favorite properties show how many favorites (for example report tables, dashboards) that have been saved in the selected period.
