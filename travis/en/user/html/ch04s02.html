4.2. Work flow

The recommended workflow is:

  1. Create organisation units.

  2. Create organisation unit groups.

  3. Create organisation unit group sets.

  4. Add organisation groups to organisation unit group sets.

  5. Add organisation units to organisation unit groups.

To avoid such situations it is recommend to first add groups to group sets, and then organisation units to groups.

It is recommend that you approach the organisation unit grouping in the following sequence (and one group set at a time):

  1. Define a new organisation unit group set, for example "Location".

  2. Add new organisation unit groups (for example "Urban", "Rural" and "Peri-urban"). Once all groups have been defined, return to the organisation unit group set and assign each of the desired groups to the group set.

  3. Go back to each group, one by one, go to edit mode and assign the organisation units that should be member of the group. Should you follow this route, you can place multiple organisation units at a time in a group. However, you must be careful not to place the same organisation units in two groups which itself is a member of an organisation unit group set. This will result in a data integrity violation. If you have organisation unit groups which are not exclusive, they should not be members of a group.

  4. A better way to ensure that you do not mistakenly assign an organisation unit to multiple members of a group set is you can use the edit feature of each organisation unit to assign memberships to each group set. You will only be able to assign a single organisation unit at a time however.
